# توثيق فلاتر منتجات التصنيفات (Products API)

تم تحديث الـ API الخاصة بالمنتجات (`GET /api/v1/products` و `GET /api/v1/public-stores/{store}/products`) لتدعم الفلاتر المطلوبة لتصنيفات المنتجات وترتيبها بشكل صحيح، مع الحفاظ على التوافق مع تطبيق Flutter.

## 1. فلترة التصنيفات (Category Filtering)
- **الباراميتر:** `category={id}`
- **التصنيفات الرئيسية (Parent Categories):** عند تمرير ID الخاص بتصنيف رئيسي (مثل Electronics = 1)، يقوم السيرفر الآن بإرجاع كافة المنتجات المرتبطة بهذا التصنيف، **بالإضافة إلى** كافة المنتجات المرتبطة بأي من تصنيفاته الفرعية.
- **التصنيفات الفرعية (Child Categories):** عند تمرير ID الخاص بتصنيف فرعي (مثل Smartphones = 2)، يقوم السيرفر بإرجاع المنتجات المرتبطة بهذا التصنيف الفرعي فقط.
- **عدد المنتجات (`products_count`):** تم تعديل نقطة النهاية `GET /api/v1/categories` ليتم جمع عدد المنتجات من التصنيفات الفرعية وإضافته إلى عدد المنتجات في التصنيف الرئيسي الخاص بها، ليعكس العدد الحقيقي الإجمالي للمنتجات المتاحة في التصنيف.

## 2. ترتيب المنتجات (Sorting)
تم دعم طريقتين للترتيب: الطريقة القديمة (بدون تحديد `sort_order`) والطريقة الجديدة (بدعم `sort_order`).

- **الباراميتر:** `sort_by`
- **الباراميتر (للترتيب التصاعدي والتنازلي):** `sort_order` (`asc` أو `desc`)

### الخيارات المتاحة لـ `sort_by`:
**الخيارات الجديدة:**
- `popular`: ترتيب حسب الأكثر شهرة (يعتمد على عدد التفضيلات `favorites_count`، عدد المبيعات `sale_count`، ومتوسط التقييم `rating_avg`).
- `price`: ترتيب حسب السعر الأساسي (`base_price`).

*ملاحظة: الخيارات الجديدة `popular` و `price` تدعم استقبال `sort_order` لترتيب تصاعدي `asc` أو تنازلي `desc`.*

**الخيارات القديمة (مدعومة للتوافق الرجعي):**
- `trending`: ترتيب حسب الأكثر تفضيلاً (تنازلياً).
- `newest`: ترتيب حسب الأحدث.
- `highest_price`: الأعلى سعراً.
- `lowest_price`: الأقل سعراً.
- `most_seller`: الأكثر مبيعاً.

*ملاحظة: الخيارات القديمة لها ترتيب افتراضي ثابت ولا تتأثر بـ `sort_order`.*

## 3. البحث والمميزات (Search & Featured)
- **الباراميتر:** `search={keyword}`
  - يمكن استخدام `search` بشكل منفرد أو مع `category`. يقوم بالبحث في عنوان المنتج ووصفه واسم المتجر.
- **الباراميتر:** `featured={true/false}`
  - يعرض المنتجات المميزة فقط.

## أمثلة على الطلبات المدعومة

**طلب منتجات تصنيف رئيسي (مع منتجات أبنائه):**
```http
GET /api/v1/products?category=1&per_page=15
```

**طلب منتجات تصنيف فرعي:**
```http
GET /api/v1/products?category=2&per_page=15
```

**طلب المنتجات الأقل سعراً في تصنيف فرعي (باستخدام الفلاتر الجديدة):**
```http
GET /api/v1/products?category=2&sort_by=price&sort_order=asc&per_page=15
```

**البحث داخل تصنيف عن كلمة معينة:**
```http
GET /api/v1/products?category=4&search=speaker&per_page=15
```

**طلب منتجات مميزة لتصنيف فرعي:**
```http
GET /api/v1/products?category=4&featured=true&per_page=15
```

## التوافقية مع Flutter
شكل الاستجابة (JSON response) لم يتغير، ويحتوي نفس هيكل `data` و `meta` المدعوم مسبقاً في التطبيق:
```json
{
  "success": true,
  "message": "Products retrieved successfully.",
  "data": [...],
  "meta": {
    "current_page": 1,
    "last_page": 1,
    "per_page": 15,
    "total": 10
  }
}
```
جميع الفلاتر وتطبيقات الترتيب تتم قبل التقسيم للصفحات (Pagination)، بحيث تعكس `meta.total` العدد الصحيح للنتائج المفلترة.
